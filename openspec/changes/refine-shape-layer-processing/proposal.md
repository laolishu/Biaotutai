## 为什么

当前 PSD 解析中，`shape` 类型图层语义过于宽泛，导入端难以直接恢复为可编辑结构，导致大量图层只能按图片处理，丢失颜色、圆角、边框等关键视觉参数。
现在需要细化 `shape` 处理策略，以提升导出 JSON 的结构化可用性，并降低对接方在导入解析阶段的二次识别成本。

## 变更内容

- 细化 `shape` 图层处理策略：
  - 优先识别为结构组件（如 `rect`/`card`）。
  - 保留颜色、圆角、边框等视觉属性并结构化输出。
  - 对复杂路径自动降级为图片资源并通过 `assetId` 关联。
- 更新导出规范文档，明确字段定义、降级策略与导入兼容要求。
- 为后续实现提供可执行的设计与任务拆解。

## 功能 (Capabilities)

### 新增功能
- `shape-layer-refinement`: 细化 `shape` 图层解析与导出规则，区分可结构化形状与复杂路径降级路径。

### 修改功能
- `layer-panel`: 图层类型与属性展示将新增细化后的 shape 语义字段（如圆角、描边、降级状态）支持。

## 影响

- 受影响模块：PSD 标准化流程、导出 JSON 结构、属性面板展示逻辑。
- 受影响文档：导出规范与对接说明。
- 兼容性：旧数据可继续解析；新策略增加可选字段，复杂路径降级为图片不破坏既有渲染路径。
