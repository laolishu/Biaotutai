## 新增需求

### 需求:Shape图层必须优先识别为结构组件
系统在解析 `shape` 图层时，必须优先尝试识别为可结构化组件（如 `rect`、`card`），并输出可直接用于导入端创建可编辑节点的结构化字段。

#### 场景:识别为矩形组件
- **当** 图层为规则矩形且可解析出边界与样式
- **那么** 系统必须将该图层输出为结构化 `shape`，并在 `props.shapeType` 中标记为 `rect`

#### 场景:识别为卡片组件
- **当** 图层满足卡片判定条件（例如存在圆角/边框且边界规则）
- **那么** 系统必须将该图层输出为结构化 `shape`，并在 `props.shapeType` 中标记为 `card`

### 需求:Shape图层必须保留关键视觉样式
系统对结构化输出的 `shape` 图层，必须保留颜色、圆角、边框等关键视觉参数，禁止仅输出未结构化样式文本。

#### 场景:输出基础样式
- **当** 图层被识别为结构化 `shape`
- **那么** 输出中必须包含可解析的 `props.fill`、`props.cornerRadius`、`props.stroke`（存在时）字段

### 需求:复杂路径必须降级为图片资源
对于无法稳定结构化的复杂路径，系统必须降级为图片资源导出，并通过 `assetId` 与 `assets` 元数据对象建立引用关系。

#### 场景:复杂路径降级
- **当** 图层包含复杂 path 或无法可靠映射为结构组件
- **那么** 系统必须导出图片资源并在图层中写入 `assetId`，且导入端可据此直接按图片渲染
