## 新增需求

### 需求:图层树从 store 渲染
左侧图层面板必须从 `store.layers` 读取真实图层数据进行渲染，禁止使用硬编码 mock 数据。

#### 场景:有图层时展示图层树
- **当** store.layers 不为空数组
- **那么** 左侧面板渲染所有顶层图层，group 类型图层默认展开并递归渲染子图层

#### 场景:无图层时展示空状态
- **当** store.layers 为空数组
- **那么** 左侧面板展示"暂无图层"提示文本

### 需求:图层可被选中
用户必须能通过点击图层面板中的图层项来选中该图层。

#### 场景:点击图层选中
- **当** 用户点击图层面板中的某个图层项
- **那么** 该图层被标记为选中状态，图层项背景高亮显示

#### 场景:选中状态与属性面板同步
- **当** 用户选中某图层
- **那么** 右侧属性面板立即更新展示该图层的属性

### 需求:图层项展示基础信息
图层项必须展示图层名称和类型图标。

#### 场景:展示图层名称与类型
- **当** 图层面板渲染某图层项
- **那么** 展示该图层的 `name` 字段文本，以及与 `type` 对应的图标（group/text/image/shape）
