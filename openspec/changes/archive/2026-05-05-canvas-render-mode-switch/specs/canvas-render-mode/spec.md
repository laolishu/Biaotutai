## 新增需求

### 需求:画布渲染模式状态管理
系统必须维护一个全局画布渲染模式状态，其值为 `'composite'`（合成图）或 `'layered'`（逐层渲染）。系统必须提供修改该状态的 action。初始值必须为 `'layered'`。

#### 场景:默认模式为逐层渲染
- **当** 用户首次加载工作台页面
- **那么** 画布渲染模式状态值为 `'layered'`，画布显示逐层渲染内容

#### 场景:切换为合成图模式
- **当** 用户调用 `setCanvasRenderMode('composite')`
- **那么** 状态更新为 `'composite'`

#### 场景:切换为逐层渲染模式
- **当** 用户调用 `setCanvasRenderMode('layered')`
- **那么** 状态更新为 `'layered'`

### 需求:双画布同时挂载
系统必须在文件加载后同时将合成背景图（`<img>` 绑定 `backgroundUrl`）和逐层渲染画布（`CanvasLayer`）挂载到 DOM 中。系统禁止在模式切换时重新生成或重新加载画布内容。

#### 场景:合成图画布在非 composite 模式下不可见
- **当** `canvasRenderMode` 为 `'layered'`
- **那么** 合成图 `<img>` 通过 `v-show="false"` 隐藏，但仍挂载在 DOM 中

#### 场景:逐层渲染画布在非 layered 模式下不可见
- **当** `canvasRenderMode` 为 `'composite'`
- **那么** `CanvasLayer` 通过 `v-show="false"` 隐藏，但仍挂载在 DOM 中

#### 场景:切换模式时无白屏
- **当** 用户切换模式
- **那么** 两种画布立即切换显隐，不出现空白过渡帧

### 需求:底部状态栏模式下拉
底部状态栏必须提供一个下拉选择控件，用于切换画布渲染模式。控件必须显示当前模式，选项为"合成图"和"逐层渲染"。

#### 场景:下拉显示当前模式
- **当** 用户查看底部状态栏
- **那么** 模式下拉控件反映当前 `canvasRenderMode` 的值

#### 场景:选择合成图
- **当** 用户在下拉中选择"合成图"
- **那么** `canvasRenderMode` 更新为 `'composite'`，合成图画布可见，逐层渲染画布隐藏

#### 场景:选择逐层渲染
- **当** 用户在下拉中选择"逐层渲染"
- **那么** `canvasRenderMode` 更新为 `'layered'`，逐层渲染画布可见，合成图画布隐藏

#### 场景:无文件时下拉不显示
- **当** 工作台尚未加载任何 PSD 文件（`store.layers` 为空）
- **那么** 模式下拉控件不渲染或禁用
