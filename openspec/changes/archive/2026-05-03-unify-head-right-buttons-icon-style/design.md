## 上下文

当前 Workbench 页头右侧操作区仍以文字按钮为主，而左右 Activity Bar 已采用图标化表达，造成视觉语言不一致与头部空间占用偏高。现有主题系统已支持 dark/light token，但在亮色模式下部分按钮边框与悬停层次对比度不足，图标按钮可辨识性受影响。

约束条件：
- 不改变既有业务行为（缩放适配、最大化、导出、主题切换）
- 不引入新依赖，继续使用 Vue + Element Plus + 现有 token 体系
- 在不破坏现有布局结构的前提下完成图标化和对齐

## 目标 / 非目标

**目标：**
- 将 Header 右侧操作统一为图标按钮，提升一致性与信息密度。
- 为每个图标按钮提供可访问语义（`title` 与 `aria-label`）。
- 统一按钮尺寸、间距、hover/active 反馈，并复用 `--color-*` token。
- 在亮色主题下提升边框和悬停层次对比度，保证图标可辨识。

**非目标：**
- 不新增或变更业务操作流程。
- 不调整左/右活动栏功能。
- 不进行全局重构或替换 UI 组件库。

## 决策

1. 采用“图标优先 + tooltip 语义补充”替代文本按钮。
- 原因：图标化可减少头部占用，并与 Activity Bar 统一视觉语言。
- 备选方案：保留文字+图标混合；放弃原因是头部密度仍偏高且风格不统一。

2. 保持交互处理函数不变，仅替换展示层。
- 原因：降低回归风险，确保行为一致。
- 备选方案：重写按钮行为与状态管理；放弃原因是改动面大、收益低。

3. 亮色模式对比度通过 token 约束提升，而非局部写死颜色。
- 原因：保证主题一致性与可维护性，避免后续页面再次出现“浅色边框不可见”。
- 备选方案：在单组件内硬编码深色边框；放弃原因是不可复用且易漂移。

4. 对“导出”操作保留强调态，其余按钮使用中性态。
- 原因：突出关键动作且不造成视觉噪音。
- 备选方案：全部强调；放弃原因是会降低主次层级。

## 风险 / 权衡

- [风险] 仅图标表达可能增加学习成本 → 缓解：补齐 `title` / `aria-label`，并保持语义化图标。
- [风险] 亮色主题调整影响暗色表现一致性 → 缓解：使用主题 token 分离 dark/light 映射，逐主题验证。
- [风险] 头部按钮尺寸统一后可能影响点击热区 → 缓解：统一最小 28px 尺寸并验证可点击区域。

## Migration Plan

1. 在 `Workbench.vue` 中将右侧文本按钮替换为图标按钮模板，保留原点击处理。
2. 在 `Workbench.vue` 样式中统一按钮尺寸、对齐与交互态，并为导出按钮设置强调态。
3. 在 `themes.css` 中强化亮色模式边框/悬停 token 对比度（如 `--color-border`、`--color-border-strong`、`--color-bg-hover`）。
4. 验证 dark/light 两个主题下视觉效果与可点击行为一致。
5. 若出现回归，回滚到上一个按钮模板与 token 映射。
