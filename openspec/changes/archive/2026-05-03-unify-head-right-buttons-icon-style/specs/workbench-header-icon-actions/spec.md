## 新增需求

### 需求:Workbench Header 右侧操作必须采用统一图标按钮
系统必须将 Workbench Header 右侧操作区按钮统一为图标按钮展示，禁止继续使用仅文本标签作为主展示形式。

#### 场景: 右侧操作区图标化展示
- **当** 用户进入 Workbench 页面
- **那么** Header 右侧操作必须以统一尺寸与样式的图标按钮呈现

### 需求:图标按钮必须保留可访问性语义
每个图标按钮必须提供可访问性语义，至少包含 `title` 与 `aria-label`，以便在无文本标签时仍可理解。

#### 场景: 鼠标悬停显示语义提示
- **当** 用户将鼠标悬停在任一图标按钮上
- **那么** 浏览器必须可显示该按钮对应的语义提示信息

### 需求:图标按钮交互行为必须与改造前一致
图标化改造必须保持既有操作行为一致，禁止因样式改造改变功能语义或事件触发链路。

#### 场景: 操作按钮行为一致
- **当** 用户点击“缩放适配”“最大化”“导出”对应图标按钮
- **那么** 系统必须触发与改造前相同的操作行为

### 需求:导出按钮必须具有主操作视觉层级
导出类按钮必须相较于其他中性按钮具备更高视觉层级，必须使用强调色或强调态进行区分。

#### 场景: 主操作可快速识别
- **当** 用户观察 Header 右侧操作区
- **那么** 用户必须能在不阅读文本的情况下快速识别导出按钮为主要操作
