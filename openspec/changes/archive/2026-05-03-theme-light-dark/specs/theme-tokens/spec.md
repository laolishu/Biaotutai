## 新增需求

### 需求:必须定义全局 CSS 变量 token 集合
系统必须在 `src/styles/themes.css` 中以 CSS 自定义属性形式定义所有颜色 token，token 名称必须以 `--color-` 为前缀，并覆盖背景、面板、画布、悬停、激活、边框、强调、文字等语义分类。

#### 场景: 暗色主题 token 生效
- **当** `<html>` 元素的 `data-theme` 属性值为 `dark`
- **那么** 所有 `--color-*` 变量必须解析为暗黑赛博朋克风格的颜色值（深蓝底色、青色强调色）

#### 场景: 亮色主题 token 生效
- **当** `<html>` 元素的 `data-theme` 属性值为 `light`
- **那么** 所有 `--color-*` 变量必须解析为仿 VS Code Light 风格的颜色值（浅灰底色、蓝色强调色）

### 需求:所有页面颜色必须引用 CSS 变量
Upload.vue、Workbench.vue、Viewer.vue 中的 `<style scoped>` 块内所有颜色声明（背景色、边框色、文字色、强调色）必须替换为对应的 `var(--color-*)` 引用，禁止保留硬编码颜色值（静态演示数据中的内联 style 除外）。

#### 场景: 主题切换时所有页面颜色更新
- **当** `data-theme` 属性在暗色与亮色之间切换
- **那么** 三个页面的所有主题相关颜色必须立即反映新主题，无需刷新页面

#### 场景: 亮色主题下发光效果降级
- **当** `data-theme` 为 `light`
- **那么** 原暗色主题的 glow `box-shadow` 效果必须以降低不透明度或去除的方式降级，避免在白色背景上显示突兀
