## 新增需求

### 需求:Upload 与 Viewer 页面必须提供主题切换入口
系统必须在 Upload 页面和 Viewer 页面都提供可直接触发主题切换的交互入口，禁止仅在 Workbench 页面提供该能力。

#### 场景:Upload 页面可直接切换主题
- **当** 用户位于 Upload 页面并触发主题切换入口
- **那么** 系统必须在 `dark` 与 `light` 之间切换，并立即更新页面主题表现

#### 场景:Viewer 页面可直接切换主题
- **当** 用户位于 Viewer 页面并触发主题切换入口
- **那么** 系统必须在 `dark` 与 `light` 之间切换，并立即更新页面主题表现

### 需求:新增入口必须复用统一主题状态源
Upload 与 Viewer 新增的主题切换入口必须复用同一全局主题状态源，禁止创建独立页面主题状态，切换后必须同步更新 `<html data-theme>` 与 `localStorage` 持久化值。

#### 场景:跨页面切换后状态一致
- **当** 用户在 Upload 页面切换主题后进入 Viewer 页面
- **那么** Viewer 页面必须保持相同主题状态，无需再次切换

### 需求:主题切换入口必须具备可访问性语义
Upload 与 Viewer 页面中的主题切换入口必须包含可访问性语义，至少提供 `title` 与 `aria-label`，禁止仅通过图标形状表达语义。

#### 场景:无可见文本时仍可理解动作
- **当** 用户在 Upload 或 Viewer 页面看到仅图标的主题切换按钮
- **那么** 浏览器必须可通过 `title` 或辅助技术读取到明确动作语义
