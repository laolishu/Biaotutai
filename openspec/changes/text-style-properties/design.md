## 上下文

右侧属性栏已经具备基础属性展示能力，但 text 类型样式字段（字体、字号、颜色）在规范层面尚未形成稳定约束，容易在后续迭代中出现字段名不一致、空值处理不统一、展示缺失等问题。需要通过本次变更明确 text 样式属性的展示与容错标准。

## 目标 / 非目标

**目标：**
- text 类型图层在右侧属性栏中必须展示字体、字号、颜色
- 当任一字段缺失时，必须有统一降级展示（如“--”）
- 非 text 类型图层禁止显示 text 样式区块

**非目标：**
- 字体编辑或颜色选择器能力
- 文本排版高级属性（行高、字重、字距）
- 非属性栏区域的 UI 调整

## 决策

1. 字段来源：以 normalize 后的 `layer.style` 为唯一来源，避免模板直接依赖原始 PSD 结构。
2. 渲染策略：属性栏使用条件渲染 `type === 'text'`，并对 `fontFamily/fontSize/color` 做逐项容错。
3. 兼容策略：保留现有基础属性与几何属性渲染，不影响 image/shape/group 分支。
4. 交互策略：继续沿用“点击复制”交互；空值不触发复制提示。

## 风险 / 权衡

- [风险] PSD 文本层样式字段不完整 → [缓解] normalize 与 UI 双层兜底，统一显示占位。
- [风险] 字号与颜色格式不统一（数值/字符串） → [缓解] 统一转为字符串渲染。
- [风险] 后续字段扩展导致模板膨胀 → [缓解] 保持 text 样式区块独立，便于扩展。
