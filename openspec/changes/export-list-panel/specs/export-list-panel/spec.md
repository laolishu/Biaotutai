## 新增需求

### 需求:导出列表（Export List）用于控制导出包含的图层
系统必须提供一个右侧 Export 列表视图，允许用户管理哪些图层将被包含在导出包中。该视图默认包含项目中所有图层并按层级组织。

#### 场景:初始化导出列表
- **当** 用户打开一个包含多层的 PSD 项目并且导出列表尚未初始化
- **那么** 系统必须基于当前图层树初始化 `exportList`，包含所有可见与隐藏的图层项，默认 `hidden=false`（即均被包含）

#### 场景:左侧图层被隐藏时同步移除
- **当** 用户在左侧图层树将某一图层或图层组设置为隐藏
- **那么** 系统必须在 `exportList` 中将对应项标记为 `hidden=true` 并在 Export 界面中立即移除该项（或以浅色/不可选状态显示，取决于 UI 决策）

#### 场景:左侧图层显示时恢复
- **当** 用户在左侧图层树将某一图层设置为显示
- **那么** 系统必须在 `exportList` 中恢复对应项为 `hidden=false`（如果未经用户 override）并重新在 Export 界面显示

#### 场景:用户在 Export 列表手动排除图层
- **当** 用户在 Export 列表中取消勾选某一图层
- **那么** 系统必须将该项的 `overridden=true` 并将 `hidden=true`，但不应改变左侧图层树的显示状态

## 修改需求

### 需求:导出行为使用 exportList 优先
现有导出逻辑必须修改为：当 `exportList` 存在且非空时，导出功能必须优先使用 `exportList` 中 `hidden=false` 的条目来生成导出包；若 `exportList` 未初始化或为空，则使用当前行为（全部图层或选中层）

#### 场景:导出时优先使用 exportList
- **当** 用户触发导出操作且 `exportList` 已初始化
- **那么** 系统必须仅包含 `exportList` 中 `hidden=false` 的图层生成导出内容
