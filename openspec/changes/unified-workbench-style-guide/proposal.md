## 为什么

标图台工作台已沉淀出一套完整的三栏式布局交互范式（Activity Bar + Activity Panel + Canvas + 右侧面板），但该设计仅以产品实现代码的形式存在，缺乏业务中立的设计规范文档。随着大屏设计器等同类产品复用相同工作台结构，若无统一的参考文档，各产品将产生风格漂移与交互不一致问题。

## 变更内容

- **新增** `docs/workbench-style-guide.md`：业务中立的工作台 UI 风格与交互规范文档
  - 覆盖布局模型（五区结构）、Activity Bar 交互模式、Activity Panel 状态、画布区、Header 操作区、拖拽调宽、视觉 token 引用
  - 以通用语言描述，屏蔽"图层/PSD/标注"等业务术语，使用"条目树/资源树"等中性命名
  - 提供接入指南：各产品如何在此框架下填充业务内容

## 功能 (Capabilities)

### 新增功能

- `workbench-style-guide-doc`：业务中立的工作台 UI 风格指南文档，为多产品复用同一工作台框架提供统一参考

### 修改功能

（无）

## 影响

- 仅影响 `docs/` 目录，新增一个 Markdown 文档
- 不修改任何源码或现有规范
- 后续接入产品（如大屏设计器）的工作台实现应参考此文档保持交互一致性
