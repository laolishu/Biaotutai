## 新增需求

### 需求:Activity Bar 图标列导航
工作台左侧必须提供一个固定宽度（44px）的 Activity Bar，纵向排列图标按钮，每个图标对应一个功能面板分页。

#### 场景: 点击未激活图标打开对应面板
- **当** 用户点击 Activity Bar 中一个未激活的图标
- **那么** 该图标进入激活状态，Activity Panel 展开并显示对应分页内容

#### 场景: 点击已激活图标收起面板
- **当** 用户点击 Activity Bar 中当前已激活的图标
- **那么** Activity Panel 收起（宽度收为 0），图标仍保持高亮，画布区域自动扩展填满

#### 场景: 初始状态
- **当** 页面首次加载
- **那么** 默认激活"图层"图标，Activity Panel 展开显示图层列表

### 需求:Activity Panel 收起与展开
Activity Panel 必须支持收起状态，收起时仅显示 Activity Bar 图标列，展开/收起过程必须有 CSS transition 动画（duration 200ms）。

#### 场景: 收起时画布扩展
- **当** Activity Panel 处于收起状态
- **那么** 中间画布区域的 flex-grow 自动占满原面板所在空间

#### 场景: 展开时恢复宽度
- **当** 用户点击图标重新展开 Activity Panel
- **那么** 面板以上次记录的宽度展开，动画平滑

### 需求:Activity Panel 右边框拖拽调整宽度
Activity Panel 右侧必须有一个可拖拽的 resize 把手，允许用户调整面板宽度，范围为 160px ~ 400px。

#### 场景: 拖拽调整宽度
- **当** 用户按下 Activity Panel 右侧边框把手并拖动
- **那么** 面板宽度实时跟随鼠标 X 轴位移变化，拖拽期间禁用 width transition 避免延迟感

#### 场景: 超出范围时限制
- **当** 拖拽导致宽度小于 160px 或大于 400px
- **那么** 宽度锁定在对应边界值，不继续缩小或放大

#### 场景: 拖拽结束
- **当** 用户松开鼠标
- **那么** 停止监听 mousemove，恢复 width transition，记录最终宽度
