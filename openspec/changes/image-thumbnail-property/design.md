## 上下文

当前右侧属性栏在 Workbench 中按选中图层展示基础属性（名称、类型、几何、文本样式），但 image 类型没有直观预览能力。用户在画布中选中图片后，需要依靠名称判断是否选中目标图片，效率偏低。

现有数据流为：PSD 解析后进入标准化图层结构，再由 store.selectedLayer 驱动右侧属性渲染。该链路可扩展 image 预览字段并在属性栏按类型条件渲染。

## 目标 / 非目标

**目标：**
- 选中 image 类型图层时，在右侧属性栏展示缩略图属性块
- 缩略图支持点击查看放大预览
- 预览资源不可用时显示占位与降级说明，不影响其他属性渲染

**非目标：**
- 图片编辑（裁剪、替换、滤镜）
- 多图对比预览
- 非 image 类型属性结构改造

## 决策

1. 数据来源决策：优先复用解析结果中的可预览来源（如图层 canvas 或已有 URL），若缺失则展示占位，不阻塞整体渲染。
2. UI 结构决策：在右侧属性栏新增 image 专属属性区块（缩略图 + 尺寸 + 状态），保持与现有属性分组样式一致。
3. 交互决策：点击缩略图触发轻量放大预览（同页浮层或简易弹层），不引入复杂依赖。
4. 容错决策：预览资源加载失败时必须回退占位图与提示文案，禁止导致属性栏崩溃或白屏。

## 风险 / 权衡

- [风险] PSD 图层可能无法直接导出单层预览 → [缓解] 先定义可选字段并做降级占位，后续逐步增强提取能力。
- [风险] 预览图加载带来额外开销 → [缓解] 使用缩略图尺寸限制与懒渲染，仅在 image 选中时加载。
- [风险] 交互复杂度上升 → [缓解] 保持预览为只读单击查看，不增加编辑功能。
