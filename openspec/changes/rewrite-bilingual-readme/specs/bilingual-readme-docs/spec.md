## 新增需求

### 需求: 英文主入口文档
项目必须提供英文 README 作为仓库默认入口文档，并在内容中清晰说明产品定位、核心能力、技术架构、页面构成与快速开始方式。

#### 场景: 访问仓库首页
- **当** 用户访问仓库首页并查看 README.md
- **那么** 看到的主文档必须为英文内容，且包含可执行的开发与构建命令说明

### 需求: 中英文文档可互相跳转
项目必须提供中文 README 文档，且英文主 README 必须包含指向中文 README 的链接；中文 README 也必须包含回到英文 README 的链接。

#### 场景: 英文读者切换中文
- **当** 用户在英文 README 中需要查看中文内容
- **那么** 文档必须提供明确可点击的中文文档链接并可成功跳转

#### 场景: 中文读者返回英文
- **当** 用户在中文 README 中需要回到主文档
- **那么** 文档必须提供明确可点击的英文文档链接并可成功跳转

### 需求: 文档内容与产品设计一致
README 中的产品描述必须与 docs/design.md 的核心定义一致，包括动态坐标系统、DOM 标注引擎、Upload/Workbench/Viewer 页面结构与阶段路线图。

#### 场景: 校验文档一致性
- **当** 维护者根据 docs/design.md 检查 README 内容
- **那么** 关键术语、功能边界与路线图表述必须保持一致，不得出现冲突描述
