## 为什么

当前仓库的 README 以中文为主，无法作为面向更广泛开发者社区的引流入口。项目已具备清晰的产品定位和技术方案（见 docs/design.md），但缺少“英文主文档 + 中文镜像文档”的双语信息架构，会降低国际读者的理解效率与转化效率。

## 变更内容

- 重写 README 文档体系，采用英文主文件作为默认入口。
- 新增中文 README 文档，并在英文主文件中以显式链接指向中文文档。
- 统一两份文档的内容结构：产品概述、核心能力、技术架构、快速开始、页面说明、路线图与贡献说明。
- 将内容与现有产品设计文档（docs/design.md）对齐，清理旧版描述中的冗余或不一致信息。

## 功能 (Capabilities)

### 新增功能
- `bilingual-readme-docs`: 项目必须提供英文主 README 与中文 README，并通过双向链接保证中英文读者可快速切换。

### 修改功能
- （无）

## 影响

- 受影响文件：README.md、新增中文 README 文件（例如 README.zh-CN.md）。
- 受影响模块：仅文档层，无运行时代码、API、依赖变更。
- 受影响流程：项目首次访问体验、开源展示与传播效率。
